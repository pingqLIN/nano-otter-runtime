Gemini Nano Local - 新手快速導覽

目標：把你本機的 Chrome OptGuide 模型包接到可互動聊天視窗，直接測試輸入/輸出。

啟動一鍵選單 (start.cmd)

提示：請透過腳本開頁面（HTTP 模式），不要直接用 file:// 開 probe/*.html。

Step 1

匯入模型包

預設來源路徑（自動偵測最新版本）：

%LOCALAPPDATA%\Google\Chrome\User Data\OptGuideOnDeviceModel\<latest_version>

也可手動輸入其他版本資料夾。

Step 2

檢查完整性

  • weights.bin
  • manifest.json
  • on_device_model_execution_config.pb

會顯示 SHA256，方便確認檔案一致性。

Step 3

啟動 Chrome + Chat Window

會建立獨立的 .chrome-user-data，不干擾你日常使用的 Chrome 設定。

啟動後會自動使用可用埠，並開啟 http://localhost:<埠號>/chat-window.html。

Step 4

確認模型模式

  • Model mode：代表已接上模型 API，可直接對話。
  • Echo mode：先驗證輸入/輸出流程，待 API 可用再切模型。

需要細查 API 時，再打開 probe/prompt-api-probe.html。

給 GitHub

模型檔禁止提交到 Git

此專案已將 model/** 設為忽略（僅保留 model/.gitkeep 與 model/README.md）。

模型來源：%LOCALAPPDATA%\Google\Chrome\User Data\OptGuideOnDeviceModel\<version>；放置位置：model/<version>/。

手動命令（進階）

powershell -ExecutionPolicy Bypass -File .\scripts\Import-OptGuideModel.ps1
# 或手動指定來源
powershell -ExecutionPolicy Bypass -File .\scripts\Import-OptGuideModel.ps1 -SourceVersionDir "C:\Path\To\OptGuideOnDeviceModel\<version>"
powershell -ExecutionPolicy Bypass -File .\scripts\Check-ModelPack.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Start-GeminiNanoChrome.ps1
